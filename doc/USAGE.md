## 1. 系统概述

本系统包含 **6 大业务模块**：

| 模块 | 功能 | 访问路径 |
|------|------|---------|
| 📊 数据看板 | 概览卡片、部门列表、考勤/审批/报销图表 | `/dashboard` |
| ⚙️ 系统管理 | 用户、角色、菜单、部门管理 | `/system/user` 等 |
| 📋 考勤管理 | 签到签退、请假申请、月度统计 | `/attendance` |
| ✅ 审批中心 | 多级审批流程、模板管理、审批撤回 | `/approval` |
| 📢 公告通知 | 发布/编辑/删除公告 | `/notice` |
| 📅 日程管理 | 日程 CRUD、日历视图、提醒通知 | `/schedule` |
| 💰 报销管理 | 报销申请、附件上传、审批联动 | `/expense` |

**三种角色**：

| 角色 | 编码 | 权限范围 |
|------|------|---------|
| 超级管理员 | `ROLE_ADMIN` | 全部数据 + 全部功能 |
| 部门经理 | `ROLE_MANAGER` | 本部门数据 |
| 普通员工 | `ROLE_EMPLOYEE` | 仅个人数据 |

## 2. 快速启动

### 2.1 后端启动（H2 开发模式）

```powershell
cd D:\CLion\oa-system
D:\CLion\tools\apache-maven-3.9.16\bin\mvn spring-boot:run
```

启动后访问：
- 应用根路径：http://localhost:8080/oa
- API 文档（在线调试）：http://localhost:8080/oa/doc.html
- H2 数据库控制台：http://localhost:8080/oa/h2-console
  - JDBC URL: `jdbc:h2:file:./data/oa-system`
  - 用户名: `sa`，密码: 留空

### 2.2 前端启动

```powershell
cd D:\CLion\oa-system\frontend
npm install   # 首次运行需要
npm run dev
```

访问：http://localhost:5173

### 2.3 重置数据库

如果遇到表结构错误（如 "Column not found"），需要删除 H2 数据库文件重建：

```powershell
Remove-Item D:\CLion\oa-system\data\oa-system.mv.db
# 然后重启后端即可自动重建
```

## 3. 菜单栏控制 —— 如何显示/隐藏功能模块

### 3.1 通过菜单管理界面控制

侧边栏的菜单项完全由 **菜单管理** 页面控制：

1. 以 **admin** 登录
2. 进入 **系统管理 → 菜单管理**
3. 在树形表格中，每个菜单项都有一个 **可见** 开关（`visible` 字段）
4. 将某菜单的 **可见** 设为 **隐藏（0）**，该菜单就会从侧边栏消失

**示例：隐藏"部门管理"菜单**
1. 系统管理 → 菜单管理
2. 找到"部门管理"这一行
3. 点击"编辑"
4. 将"是否可见"切换为 **隐藏**
5. 保存后刷新页面，侧边栏不再显示"部门管理"

> **注意**：这只是隐藏菜单入口，不删除数据。部门管理 API 仍然可用，你仍然可以通过直接访问 URL `/system/dept` 进入。

### 3.2 通过数据库直接控制

你也可以直接通过 H2 控制台修改 `sys_menu` 表：

```sql
-- 隐藏"部门管理"菜单
UPDATE sys_menu SET visible = 0 WHERE id = 5;

-- 隐藏"报销管理"整个模块(含子菜单)
UPDATE sys_menu SET visible = 0 WHERE id = 50;  -- 目录
UPDATE sys_menu SET visible = 0 WHERE parent_id = 50;  -- 子菜单

-- 恢复显示
UPDATE sys_menu SET visible = 1 WHERE id = 5;
```

### 3.3 菜单表字段说明

| 字段 | 含义 | 说明 |
|------|------|------|
| `menu_type` | 类型 | 0=目录（分组），1=菜单（页面路由） |
| `visible` | 可见性 | 0=隐藏，1=显示 |
| `status` | 状态 | 0=禁用，1=启用 |
| `perms` | 权限标识 | 如 `system:dept:list`，用于后端权限控制 |
| `sort` | 排序 | 数字越小越靠前 |

### 3.4 给不同角色分配不同菜单

1. **系统管理 → 角色管理**
2. 选择某个角色，点击 **分配菜单**（或操作列中的权限按钮）
3. 在树形列表中勾选该角色能看到的菜单
4. 保存

例如：给"普通员工"角色只分配"数据看板"、"考勤管理"、"日程管理"，不分配"系统管理"和"审批中心"——该角色登录后就只能看到被分配的菜单。

## 4. 部门管理

### 4.1 查看部门

**数据看板** 的"部门列表"卡片展示了所有部门及其人数统计。  
**系统管理 → 部门管理** 展示了完整的部门树，包含负责人、本部门人数、总人数（含下级）。

### 4.2 新增部门

1. 进入 **系统管理 → 部门管理**
2. 点击 **新增部门** 按钮 → 创建顶级部门（母公司级别）
3. 或点击某部门的 **新增子** 按钮 → 创建该部门的下级部门
4. 填写部门名称、选择负责人（可选）、排序号会自动递增
5. 点击确定

### 4.3 设置部门负责人

在新增/编辑部门的对话框中，**负责人** 下拉框列出了所有用户，选择即可。

部门负责人是该部门的默认审批人——当员工提交请假/报销时，第一级审批会自动路由到其部门负责人。

### 4.4 隐藏（禁用）部门

部门管理页面的"状态"列显示启用/禁用：
- **禁用**（status=0）：部门不会出现在下拉选项中，但已有数据保留
- 编辑部门 → 将状态切换为禁用 → 保存

### 4.5 删除部门

点击删除按钮即可。**注意**：删除部门不会自动删除其子部门或下属用户，请确认无关联数据后再删除。

## 5. 用户管理

### 5.1 查看用户列表

**系统管理 → 用户管理**：表格展示所有用户的用户名、真实姓名、所属部门、工号、联系方式等。

### 5.2 新增/编辑用户

1. 点击 **新增用户**
2. 填写用户名、密码、真实姓名
3. **部门** 下拉框以树形展示所有启用的部门，选择用户所属部门
4. 设置状态（启用/禁用）
5. 保存

### 5.3 重置密码

点击 **重置密码** 按钮，密码将重置为 `123456`。

### 5.4 分配角色

用户管理页面当前不直接支持分配角色（后续可通过角色管理页面或专门的用户-角色分配功能来实现）。

## 6. 角色与权限管理

### 6.1 角色 CRUD

**系统管理 → 角色管理**：查看/新增/编辑/删除角色。

关键字段：
- **角色编码**：如 `ROLE_ADMIN`、`ROLE_MANAGER`，用于后端 `@PreAuthorize` 注解
- **数据范围**：1=全部数据，2=本部门及子部门，4=仅本人

### 6.2 给角色分配菜单权限

点击角色的 **分配菜单** 按钮，在弹出树中勾选该角色能访问的菜单。保存后，拥有该角色的用户登录时只能看到被分配的菜单项。

### 7.1 概览卡片

顶部 4 个卡片展示：用户总数、部门总数、角色总数、公告总数。

### 7.2 部门列表

展示所有部门名称及其人数统计（本部门人数 / 含下级总人数）。

- **报销类型分布**：差旅/办公/招待/交通/其他的柱状图+折线图

### 7.4 快捷入口

6 个模块快捷入口卡片，点击可跳转到对应管理页面。

### 7.5 日程提醒

如果设置了日程提醒，数据看板会展示未来 24 小时内即将到来的日程。

## 8. 审批中心

### 8.1 审批流程说明

审批流程基于 **审批模板**：
1. **模板** 定义了审批级别和每级的审批人类型
2. **发起审批** 时，系统根据模板自动解析审批人并创建审批记录
3. 审批人依次审批，同意→推进下一级，驳回→流程结束
4. 所有级别通过后，关联的业务记录（请假/报销）自动更新状态

### 8.2 审批人类型

| 类型 | 说明 |
|------|------|
| `DEPT_LEADER` | 申请人的部门负责人 |
| `ROLE` | 拥有指定角色的所有用户（如 ROLE_ADMIN） |
| `USER` | 指定的具体用户 |

### 8.3 并行审批（DEV-30）

模板中同一级别可以配置多个审批人，任一同意即推进，任一驳回则结束。

### 8.4 撤回

申请人可以在审批完成前撤回自己的申请。

## 9. 日程管理

### 9.1 列表视图

按日期范围筛选所有日程，显示标题、时间、类型、优先级、状态。

### 9.2 日历视图

点击 **日历视图** 按钮切换到 FullCalendar 视图：
- 月/周/日三种视图
- 点击空白日期快速创建日程
- 点击事件编辑日程
- 不同颜色区分：蓝色=个人、绿色=部门、橙色=会议

### 9.3 可见性设置

| 可见性 | 谁能看到 |
|--------|---------|
| 仅自己 | 只有创建者本人 |
| 本部门 | 同部门的同事 |
| 公开 | 所有人可见 |

### 9.4 提醒设置

可设置提前 5分钟 ~ 1天提醒。设置后，数据看板的"近期日程提醒"区域会展示。

## 10. 常见问题与排错

### 10.1 启动报 "Column not found" / 表结构相关错误

**原因**：H2 数据库文件是用旧版 schema 创建的，缺少新增字段。

**解决**：
```powershell
Remove-Item D:\CLion\oa-system\data\oa-system.mv.db
# 重启后端
```

### 10.2 前端页面显示"系统繁忙"

**排查步骤**：
1. 查看后端控制台日志，找到完整的异常堆栈
2. 常见原因：
   - H2 数据库字段缺失（参考 10.1）
   - 后端未启动
   - 前端代理配置错误

### 10.3 登录后看不到某些菜单

1. 检查你的角色是否被分配了对应菜单（系统管理 → 角色管理 → 分配菜单）
2. 检查菜单的 `visible` 字段是否为 1（显示）
3. 检查菜单的 `status` 字段是否为 1（启用）

### 10.4 新增部门/日程时报错

1. 首先确认 H2 数据库已重建（参考 10.1）
2. 确认你以 admin 登录（部门管理需要 ROLE_ADMIN 权限）
3. 查看 API 文档（http://localhost:8080/oa/doc.html）在线调试确认接口正常

### 10.5 如何不显示部门的某些信息

**隐藏整个部门管理页面**：菜单管理中将"部门管理"设为隐藏

**不在数据看板中显示部门列表**：修改 `frontend/src/views/dashboard/index.vue`，删除"部门列表"的 `<el-row>` 区块

**禁止普通用户查看部门**：不给其角色分配"部门管理"菜单权限

### 10.6 忘记 admin 密码

1. 打开 H2 控制台
2. 执行：
```sql
UPDATE sys_user SET password = '$2a$10$N.zmdr9k7uOCQb376NoUnuTJ8iAt6Z5EHsM8lE9lBOsl7iKTVKIUi'
WHERE username = 'admin';
```
3. 密码重置为 `123456`
